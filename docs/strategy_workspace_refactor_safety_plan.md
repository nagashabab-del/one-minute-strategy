# Strategy Workspace Refactor Safety Plan

## الهدف
تقليل مخاطر ملف `app/strategy-workspace.tsx` الكبير عبر خطة تدريجية لا تغيّر السلوك الوظيفي.

## دليل الحالة الحالية
- حجم الملف: `15877` سطر.
- مؤشرات التعقيد:
  - حالات `useState` كثيرة تبدأ تقريبًا من السطر `1419`.
  - اعتماد مباشر على `localStorage` في عدة نقاط (مثل `983`, `1006`, `1028`, `1054`, `1705`, `1785`).
  - كثافة أحداث واجهة (`onClick`) عالية جدًا في النطاق ~`10000` إلى `15800`.
- ملاحظة Babel الحالية:
  - `deoptimised the styling ... as it exceeds the max of 500KB`

## المخاطر الرئيسية
1. كسر ترتيب/نطاق الـ`hooks` عند نقل المنطق الحي.
2. فقدان تماسك البيانات بين الواجهة والتخزين المحلي.
3. تغيّر غير مقصود في السلوك بسبب نقل `useEffect` أو dependencies.
4. فرق SSR/CSR عند إدخال lazy/dynamic import بشكل غير مضبوط.

## قواعد عدم الكسر (Guardrails)
1. لا يتم نقل `state`, `useEffect`, `handlers` في المرحلة الأولى.
2. أي دفعة تعديل يجب أن تكون صغيرة ومحدودة في نطاق واحد فقط.
3. كل دفعة تتطلب تحققًا إلزاميًا:
   - `npm run lint`
   - `npm run build` (عند تغيير هيكل كبير)
   - فحص UI يدوي لمسارات أساسية.
4. لا حذف لمسارات أو تغيير أسماء مفاتيح `localStorage` بدون طبقة توافق.

## خطة التنفيذ التدريجية
### المرحلة 1 (آمنة جدًا)
- استخراج الثوابت والأنواع الصرفة فقط إلى ملفات `lib` منفصلة.
- لا تغيير في السلوك، ولا نقل logic حي.

### المرحلة 2 (آمنة)
- استخراج الدوال النقية (`pure utils`) التي لا تعتمد على React state.
- الإبقاء على الاستدعاء من الملف الأصلي لضمان نفس السلوك.

### المرحلة 3 (متوسطة المخاطر)
- استخراج أجزاء واجهة Presentation-only لا تملك state خاص.
- تمرير البيانات عبر props فقط.

### المرحلة 4 (أعلى مخاطر - لاحقًا)
- تقييم استخراج hooks منطقية تدريجيًا مع اختبارات أدق.
- هذه المرحلة لا تبدأ إلا بعد ثبات المرحلتين 1 و2.

## معايير إيقاف فوري
1. أي regression في مسار التشغيل الأساسي.
2. اختلاف نتائج التحليل/التقرير لنفس الإدخالات.
3. أخطاء hydration أو runtime جديدة.

## خطة تراجع مختصرة
1. عزل كل دفعة في commit مستقل صغير.
2. عند ظهور خلل: `git revert <commit>` للدفعة المتأثرة فقط.
3. إعادة الاختبار بعد التراجع قبل متابعة أي دفعة جديدة.

## القرار الحالي
- لا يتم تنفيذ refactor فعلي الآن.
- يتم اعتماد هذه الخطة كمرجع أمان قبل أي تعديل مستقبلي على الملف الكبير.

# عدم إظهار مراحل الاستراتيجية بدون مشروع فعلي

التاريخ: 2026-03-09

## الهدف
تطبيق قاعدة واضحة: لا تظهر مراحل الاستراتيجية إلا عند وجود مشروع فعلي.

## السبب
كان النظام يتعامل مع مشروع افتراضي قديم (`local_default_project`) كمشروع نشط، فيظهر الشريط المرحلي حتى قبل إنشاء مشروع حقيقي.

## الملفات المعدلة
- `app/app/strategy/_lib/readiness.ts`
- `app/app/strategy/layout.tsx`
- `app/app/shell.tsx`

## ما تم
1. في `readiness.ts`:
   - تمييز المشروع الافتراضي القديم كحالة "لا مشروع فعلي" إذا كان بدون بيانات حقيقية.
   - إرجاع مشروع `global` عند عدم وجود مشروع فعلي.
   - جعل `readActiveStrategyProjectId()` يعيد قيمة فارغة بدل `global` لتفادي الاعتماد الخاطئ.

2. في `strategy/layout.tsx`:
   - إخفاء قائمة مراحل الاستراتيجية عندما لا يوجد مشروع فعلي.
   - إظهار لوحة بديلة واضحة: "لا يوجد مشروع نشط" + زر "فتح مركز المشاريع".

3. في `shell.tsx`:
   - مزامنة حالة وجود مشروع فعلي.
   - عند عدم وجود مشروع:
     - حالة البيانات تصبح "بدون مشروع".
     - الإجراء الرئيسي يصبح "فتح مركز المشاريع".
     - إخفاء اختصارات تعتمد على مشروع فعلي (مثل التقارير).

## نتيجة المستخدم
- إذا لا يوجد مشروع فعلي: لن تظهر مراحل الاستراتيجية.
- يظهر مسار صحيح لإدارة المشاريع أولاً ثم دخول المراحل.

## التحقق
- `npm run lint`: ناجح.
- `npm run build`: ناجح.

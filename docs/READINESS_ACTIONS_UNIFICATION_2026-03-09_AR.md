# توحيد إجراءات جاهزية البيانات

التاريخ: 2026-03-09

## الهدف
توحيد منطق قفل الإجراءات في وضع فجوة البيانات داخل نقطة مركزية واحدة، بدون تغيير سلوك الواجهة.

## النطاق
- لا توجد إضافة ميزات جديدة.
- لا يوجد تغيير في تدفق المستخدم.
- التغيير يخص توحيد منطق القرار فقط.

## الملفات المتأثرة
- `app/app/_lib/readiness-actions.ts`
- `app/app/page.tsx`
- `app/app/workflows/page.tsx`
- `app/app/shell.tsx`
- `app/app/reports/page.tsx`
- `app/app/reports/[id]/report-details-client.tsx`

## ما تم تنفيذه
1. إنشاء ملف مركزي جديد:
   - `app/app/_lib/readiness-actions.ts`
   - يوفر دوال موحدة:
     - `isReadinessActionBlocked`
     - `resolveReadinessNavigationHref`
     - `resolveReadinessBlockedHint`

2. استبدال التكرار في الصفحات المذكورة:
   - نقل شرط القفل من كل صفحة إلى الدوال المركزية.
   - توحيد مسار التحويل في وضع الفجوة إلى:
     - `/app/strategy/brief`
   - توحيد نص التلميح الخاص بالقفل عبر نقطة واحدة.

## التحقق
- `npm run lint`: ناجح.
- `npm run build`: ناجح.
- ملاحظة غير حرجة من Babel على ملف كبير:
  - `app/strategy-workspace.tsx`
  - ليست خطأ بناء.

## المخاطر
- مخاطرة منخفضة؛ التغيير تجميعي لمنطق موجود مسبقًا.
- لا تعديل في شكل الواجهة أو العقود البرمجية العامة.

## نتيجة الدفعة
- أصبح قرار القفل/التحويل موحدًا في نقطة واحدة.
- تقليل تكرار الشروط داخل الصفحات التشغيلية.

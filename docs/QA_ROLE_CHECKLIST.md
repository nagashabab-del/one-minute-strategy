# QA سريع للصلاحيات (v1.1)

## الهدف
التأكد أن كل دور يرى نفس البيانات، لكن لا يعدّل إلا ما تسمح به صلاحياته.

## طريقة الاختبار
1. افتح النظام.
2. من أعلى الصفحة غيّر `الدور الحالي`.
3. نفّذ الحالات التالية وتأكد من النتيجة المتوقعة.

## مدير مشروع
1. عدّل إعدادات الجلسة وبيانات المشروع والميزانية.
2. أكمل الأسئلة وشغّل التحليل.
3. ادخل المسار المتقدم وعدّل `BOQ` و`Action Tracker`.
4. نفّذ `Freeze` وأنشئ `Change Request`.
5. فعّل `اعتماد نهائي للخطة` ثم `حفظ واعتماد`.
6. النتيجة المتوقعة: كل الإجراءات متاحة.

## عمليات
1. عدّل إعدادات الجلسة وبيانات المشروع.
2. أكمل الأسئلة وشغّل التحليل.
3. عدّل حقول المسار المتقدم و`Action Tracker`.
4. جرّب `Freeze` أو `إنشاء طلب تغيير`.
5. جرّب `حفظ واعتماد`.
6. النتيجة المتوقعة: التنفيذ متاح، الحوكمة والاعتماد غير متاحين.

## مالي
1. جرّب تعديل إعدادات الجلسة أو بيانات التنفيذ المتقدم.
2. جرّب تعديل الميزانية.
3. ادخل المسار المتقدم وجرّب `Freeze` و`Change Request`.
4. جرّب `حفظ واعتماد`.
5. النتيجة المتوقعة: الميزانية + الحوكمة متاحة، التنفيذ والاعتماد غير متاحين.

## مشاهد فقط
1. مرّ على كل المراحل.
2. جرّب كل الأزرار الأساسية.
3. النتيجة المتوقعة: عرض فقط، تظهر رسائل توضح سبب المنع.

## قبول الإصدار
1. لا تظهر أخطاء TypeScript أو ESLint.
2. لا يحدث تعارض بين رسائل النجاح والخطأ.
3. تظهر رسالة `قيود الصلاحية في هذه المرحلة` عند أي منع صلاحية.
